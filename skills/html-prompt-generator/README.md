# html-prompt-generator

根据需求概述，生成可直接投喂给代码生成模型的高质量中文 HTML 提示词。

## 用途

当你有一个模糊的需求，想让它变成结构清晰、可直接使用的提示词时，用这个技能。它不直接写代码，而是产出提示词文本。

适用场景：PPT 风格网页、官网首页、数据看板、落地页、教学 Demo、交互原型等。

## 使用方式

```
/html-prompt-generator
/html-prompt-generator 帮我做一个电商首页的提示词
/html-prompt-generator 做一个数据看板，深色主题
```

## 工作流程

### 1. 信息确认

技能会先检查需求是否完整，缺失时逐步询问。确认以下内容：

- 目标产物类型
- 主题 / 业务目标
- 功能要求（至少 2-3 条）
- 风格要求
- 技术偏好
- 交付要求
- 图标库偏好
- 响应式策略

### 2. 生成提示词

默认输出 **3 个**差异化定位的提示词：

| 版本 | 定位 | 适合场景 |
|------|------|---------|
| 提示词 A（稳健版） | 结构清晰、约束完整 | 一次成稿，不迭代 |
| 提示词 B（平衡版） | 质量与创意平衡 | 大多数项目 |
| 提示词 C（探索版） | 鼓励视觉表达与交互创新 | 追求差异化 |

每个提示词包含：角色定义、任务目标、内容规划、设计风格、技术要求、资源依赖、交付标准、可验收条件。

## 输出格式

```markdown
## 信息确认摘要
- 目标产物：...
- 主题：...
- ...

## 提示词 A（稳健版）
[可直接复制使用的完整提示词]

## 提示词 B（平衡版）
[可直接复制使用的完整提示词]

## 提示词 C（探索版）
[可直接复制使用的完整提示词]
```

## 示例

**输入：**
```
做一个 SaaS 产品落地页，蓝色科技风
```

**交互过程：**
```
> 功能要求有哪些？A) 特性介绍+定价+CTA B) 仅展示特性 C) 自定义
> 图标库偏好？A) Font Awesome B) Material Symbols C) Heroicons D) 不指定
> 响应式策略？A) 桌面优先 B) 移动优先 C) 双端都重点
```

**输出：** 三个完整提示词，可直接分别投喂代码生成模型，对比效果。

## 注意事项

- 技能只产出提示词，不直接输出代码
- 全程中文交互
- 用户可以要求只生成 1 个或 2 个版本
- 如果用户提供了自己的模板格式，技能会沿用其字段顺序与术语
